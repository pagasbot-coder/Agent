## 1. Что считаем «активным пользователем»

Сценарий **WAU** (weekly active user) — PM, который реально пользуется co-pilot:

| Действие | Вызов LLM? | Частота (база) |
|----------|:----------:|----------------|
| Онбординг (4 шага) | **1×** при первом заходе на дашборд | 1 раз в жизни аккаунта |
| **Обновление HealthCommentary** (кнопка «Обновить» / смена контекста) | **Да** | **N = 4 раза/нед** (база) |
| Авто-загрузка при открытии дашборда | **Да** | **1 раз/нед** (отдельная сессия) |
| «Фокус недели» (T-080) | **Нет** | статические вопросы |
| Чек-лист онбординга, стейкхолдеры, снимок недели | **Нет** | localStorage / static |

**Базовый месяц:** ~**20 вызовов LLM** (= 4 обновления × 4 нед + 4 авто-открытия).  
Онбординг (+1 вызов) учитываем отдельно только в **первом месяце**.

## 2. Оценка токенов на один вызов

Модель: **`deepseek-chat`** (BFF, ADR-001) — по документации DeepSeek (июнь 2026) маппится на **V4-Flash** (non-thinking).

| Компонент | Токены (оценка) | Источник |
|-----------|----------------:|----------|
| System prompt (`getSystemPrompt`) | **550–650** | ~1 100 символов RU |
| User message (8 доменов + meta + JSON-инструкция) | **450–550** | `buildUserMessage()` |
| Опционально Navigator (`userSituation`) | **+50–120** | сценарии S1–S4 |
| **Вход (input), типично** | **~1 000–1 100** | |
| **Ответ (output), cap** | **512 max** (`ADVISOR_MAX_TOKENS`) | |
| **Ответ, типично** | **180–280** | 2–4 предложения + 1–3 вопроса JSON |
| **Итого на сессию (биллинг)** | **~1 200–1 400** | input + output |

> **Консервативная модель для расчётов:** **1 050 input + 250 output = 1 300 токенов/вызов** (+20% буфер по T-060 → **~1 560 токенов** в чувствительности).

## 5. Два режима DeepSeek

### A. Баланс $0 (текущий prod, июнь 2026)

| Параметр | Значение |
|----------|----------|
| **API cost** | **$0** |
| **Поведение** | HTTP 402 / insufficient balance → **fallback RU** + суффикс «офлайн-режим» |
| **Что работает без LLM** | DomainRadar, онбординг, фокус недели, стейкхолдеры, снимок, waitlist |
| **Что деградирует** | Живой co-pilot: нет персонализированных вопросов от модели |

### B. Баланс пополнен (live co-pilot)

| Интенсивность | Вызовов/мес | USD/мес | ~RUB/мес |
|---------------|------------:|--------:|---------:|
| **Лёгкий** (10 refresh) | ~12* | $0,003 | **~0,2 ₽** |
| **База** (N=4/нед) | ~20 | $0,005 | **~0,4 ₽** |
| **Активный** (50 refresh) | ~55* | $0,014 | **~1,0 ₽** |
| **Тяжёлый** (100 refresh) | ~105* | $0,027 | **~2,0 ₽** |

\* С учётом ~2 авто-открытий/нед; без онбординга.

**Вывод:** даже **100 обновлений/мес** — порядка **2 ₽ API** vs **990 ₽** гипотезы Pro.

## 6. Сравнение с Pro ~990 ₽/мес (napkin margin)

| Статья | База (20 вызовов) | Тяжёлый (100) |
|--------|------------------:|--------------:|
| Выручка Pro (гипотеза) | 990 ₽ | 990 ₽ |
| API DeepSeek | ~0,4 ₽ | ~2 ₽ |
| **Валовая маржа на API** | **~99,96%** | **~99,8%** |
| Эквайринг YooKassa (~3%)* | ~30 ₽ | ~30 ₽ |
| **После эквайринга (napkin)** | **~960 ₽** | **~958 ₽** |

\* Оценка комиссии; точный тариф — в договоре merchant YooKassa.

**Не в этой модели (но важно для P&L):** Vercel, Neon Postgres, PostHog, Human support — см. T-060 / infra отдельно.  
Целевой ориентир из landing one-pager: API **&lt;$0,50/user/week** — **фактический baseline в ~100× дешевле**.

## 7. Guardrails в приложении (уже включены)

| Механизм | Значение по умолчанию | Эффект |
|----------|----------------------|--------|
| **Rate limit** | 20 req / 15 min / IP (или user при AUTH) | Защита от burst; max ~1 920 req/сутки теоретически |
| **ADVISOR_MAX_TOKENS** | 512 | Потолок output на вызов |
| **ADVISOR_DAILY_TOKEN_BUDGET** | 0 (выкл.) | При &gt;0 — fallback после лимита |
| **ADVISOR_WEEKLY_TOKEN_BUDGET** | **200 000** | ~**150 вызовов/нед** на инстанс → fallback, не 500 |
| **Redis rate limit** (T-036) | OFF без `REDIS_URL` | In-memory на Vercel; per-user — после AUTH |
| **Fallback** | RU static | Нет 500; нет утечки ключей |

См. [`costGuardrails.ts`](../lib/advisor/costGuardrails.ts) · `/api/health` → `cost_guardrails` snapshot.

## 8. Чувствительность: 10 / 50 / 100 обновлений в месяц

Предположение: каждое «обновление» = 1 POST `/api/advisor/health-commentary`, ~1 300 токенов, тариф V4-Flash.

| Обновлений/мес | Вызовов всего* | Токенов/мес | USD/мес | ~RUB/мес | % от 990 ₽ Pro |
|----------------|---------------:|------------:|--------:|---------:|---------------:|
| **10** | 12 | 15,6K | $0,003 | **0,2 ₽** | 0,02% |
| **50** | 55 | 71,5K | $0,014 | **1,0 ₽** | 0,10% |
| **100** | 105 | 136,5K | $0,027 | **2,0 ₽** | 0,20% |

\* + авто-открытия дашборда (~2/нед).

**+20% сценарий (T-060):** умножить RUB-столбец на **1,2** → макс. **~2,4 ₽** при 100 refresh.
