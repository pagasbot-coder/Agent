## Цель

**Time to first radar < 3 минут** — пользователь проходит короткий опрос и видит заполненный DomainRadar без ручного ввода всех 8 доменов.

---

## Шаги (4 экрана)

Прогресс: **1/4 → 2/4 → 3/4 → 4/4**. Подписи шагов: **Профиль · Турбулентность · Проработка · Боль**.

### Шаг 1 — Профиль проекта

| Элемент | Тип | Варианты |
|---------|-----|----------|
| Название проекта | text | placeholder «Например: Релиз CRM» |
| Подход к поставке | radio | **Предиктивный** · **Адаптивный** · **Гибридный** |

**Ценность:** tailoring (PMBOK Principle 7) — влияет на подсказки HealthCommentary.

**Маппинг:** `deliveryApproach` → store.

---

### Шаг 2 — Турбулентность (слайдеры 1–5)

| Вопрос (RU) | Влияет на домены |
|-------------|------------------|
| Насколько нестабильны **требования**? | D4, D6 |
| Насколько нестабильны **технологии**? | D5, D8 |
| Насколько сложны **стейкхолдеры**? | D1, D8 |

Формула: `score = 100 - (slider * 15)` с clamp 15–85.

**Кнопка:** «Далее» → шаг 3. **«Назад»** → шаг 1.

---

### Шаг 3 — Проработка проекта (T-081)

Чек-лист «что уже ясно» — **7 пунктов**, multi-select. Можно **«Пропустить»** (без бонусов к scores).

| ID | Пункт (RU) | Бонус к доменам (если отмечено) |
|----|------------|----------------------------------|
| `goal` | Цель проекта сформулирована одной фразой | D4 +8, D6 +8 |
| `boundaries` | Понятно, что НЕ входит в проект | D4 +6, D8 +5 |
| `stakeholders_named` | Ключевые стороны названы | D1 +10 |
| `success_4w` | Критерий успеха на 4 недели есть | D6 +7, D7 +5 |
| `top_risk` | Главный риск записан | D8 +10 |
| `sponsor_clear` | Есть понятный спонсор/заказчик | D1 +8 |
| `business_value` | Ценность для бизнеса понятна команде | D1 +5, D6 +5 |

**Реализация:** `ProjectPrepChecklistStep` · `PREP_CHECKLIST_ITEMS` в `lib/onboarding.ts` · `computePrepChecklistScoreDelta()`.

**Кнопки:** «Далее» (с выбранными пунктами) · «Пропустить» · «Назад» → шаг 2.

---

### Шаг 4 — Главная боль (чекбокс, один выбор)

| Вариант | Буст проблемных доменов |
|---------|-------------------------|
| Сроки и план | D4 −15, D6 −10 |
| Люди и команда | D2 −20 |
| Заказчик и стейкхолдеры | D1 −20, D8 −10 |
| Риски и неизвестное | D8 −25 |
| Метрики и отчёты | D7 −15 |

**Кнопка:** «Показать радар» → `hydrateFromOnboarding()` → `/` dashboard.

**Порядок score mapping:** baseline 70 → турбулентность (шаг 2) → prep bonuses (шаг 3) → pain penalties (шаг 4).

## Глоссарий 8 доменов (hover / «?»)

| ID | Заголовок | Подсказка (1 строка) |
|----|-----------|----------------------|
| D1 | Заинтересованные стороны | Кто влияет на проект и согласованы ли ожидания |
| D2 | Команда | Роли, нагрузка, психологическая безопасность |
| D3 | Подход и жизненный цикл | Waterfall, Agile или гибрид на практике |
| D4 | Планирование | Живой ли план на 2–3 недели вперёд |
| D5 | Работа проекта | Поток задач и ресурсов |
| D6 | Поставка | Принимается ли ценность заказчиком |
| D7 | Измерение | Метрики ведут к решениям, а не к отчёту |
| D8 | Неопределённость | Риски и «мы не знаем» под контролем |

На шаге 4 (боль) — тот же `DomainGlossary`, что на dashboard (T-037).
