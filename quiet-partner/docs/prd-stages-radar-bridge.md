## TL;DR

Два режима остаются **сами по себе**. Связь — односторонняя и явная: заполнил проект в пульте → **подтянул в Тихого напарника** → на радаре видишь **этот проект и его оценку** (8 доменов + комментарий). Без комбайна UI, без скрытого автомержа, без второго источника правды «непонятно кто канон».

---

## 1. Принцип объединения (аккуратно)

| Слой | Владелец | Где живёт | Остаётся автономным? |
|------|----------|-----------|----------------------|
| **Операционка** — этап 0–6, реестры, MD-экспорт | Пульт `/stages` | `qp-stages-*` localStorage | Да — можно пользоваться только пультом |
| **Рефлексия** — 8 доменов, AI, фокус дня/недели | Напарник `/radar` | Zustand project store | Да — можно пользоваться только радаром |
| **Мост** — связь + оценка | Явное действие «Подтянуть в напарника» | общий `link` + снимок | Не третий продукт |

**Канон v1**

- Таблицы и факт «где мы по этапу» → **пульт**.
- Оценка здоровья доменов и AI → **напарник**.
- Мост **копирует снимок** из пульта в store напарника и предлагает/ставит **оценку** (см. §3). Не делает радар редактором реестров.

**Не делаем**

- Один экран «всё сразу» (комбайн).
- Тихий автосинк при каждом кейстроке без подтверждения (шум + сюрпризы).
- Автомаппинг реестров → «жёсткие» скоры без правил Senior PM и без возможности поправить руками.
- File System Access / серверный sync как обязательный путь (Phase 5 / ADR отдельно).
- Waitlist как третий режим на hub.

---

## 2. User stories

1. Как PM, я веду реестры только в пульте; радар мне не обязателен.
2. Как PM, после наполнения пульта я жму **«Подтянуть в Тихого напарника»** и на `/radar` вижу **имя этого проекта** и **радар с оценкой**.
3. Как PM, я понимаю, что оценка — **со-пилот** (disclaimer), могу подвигать скоры руками.
4. Как PM на dogfood, «Тестовый прогон» одним жестом наполняет пульт **и** даёт оценку в напарнике (опциональный чекбокс / вторая кнопка).

---

## 3. Оценка: что значит «показал его оценку»

После подтягивания на `/radar`:

| Элемент | Поведение v1 |
|---------|----------------|
| Имя проекта | = имя из пульта |
| Этап / фаза | отображается (stage 0–6 → `phase` / баннер «из пульта») |
| DomainRadar | scores из **правил маппинга** Senior PM (T-105) поверх baseline, либо «мягкие» подсказки + apply |
| HealthCommentary | обычный BFF по новым scores |
| Фокус дня | опционально предложить из слабого домена (reuse `buildRadarFocusDay`) |

**Режим оценки (решение по умолчанию для v1):**

- **A (рекомендация PM):** при «Подтянуть» сразу **применить suggested scores** в store + toast «Оценку можно поправить на радаре». Быстрее dogfood «увидел оценку».
- **B (осторожнее):** показать превью «предложенная оценка» → кнопка «Применить».

Human выбирает A или B при Go на T-106/T-107. По умолчанию в очереди заложено **A**, с confirm если в напарнике уже есть другой проект с другим именем.

## 4. Контракт данных (черновик → ADR T-104)

```text
stagesRadarBridge v1
────────────────────
link: {
  projectKey: string      // стабильный id в браузере
  linkedAt: ISO
  source: "stages"
  stagesUpdatedAt: ISO    // для «устарел ли снимок»
}
project: { name, stageId: 0..6 }
registers: { …краткая выжимка или полный cache как в пульте… }
suggestedScores: Record<DomainId, number>  // из правил T-105
appliedScores?: boolean
```

Поток:

1. Пульт: CTA **«Подтянуть в напарника»** → собирает snapshot из `qp-stages-*`.
2. Пишет/мержит в Zustand: `projectProfile.name`, phase/stage, domains, `bridgeLink`.
3. Redirect или deep-link `/radar?from=stages` с баннером «Проект из пульта».
4. Обратный поток (напарник → пульт) **out of scope v1**.

Существующий `exportProjectSnapshot` (только домены) **не ломаем**; мост — отдельный тип или расширение с `schemaVersion`.

## 5. Out of scope (v1)

- Multi-project desk / список проектов на сервере  
- Автосинк при каждой правке ячейки  
- Редактирование реестров на `/radar`  
- Auth, Postgres project rows  
- Склейка FocusWeek и FocusDay в одну карточку  

---

## 6. Success metrics

- Dogfood: «заполнил пульт → за ≤30 сек увидел оценку на радаре» без объяснения.  
- Оба режима открываются и работают **без** моста (регресс автономности).  
- PostHog (если ON): `bridge_pull_to_radar`, `bridge_scores_applied`.

---

## 7. Риски

| Риск | Митигация |
|------|-----------|
| Два имени проекта «разъехались» | Confirm при overwrite; баннер `stagesUpdatedAt` |
| Ложная точность скоров | Disclaimer + Senior PM rules + ручная правка |
| Пользователь думает, что режимы слились | Copy: «связь», не «объединение»; Mode Hub без изменений смысла |
