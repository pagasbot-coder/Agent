# Круглый стол: оценка ProjectM + объединение с Quiet Partner

**Дата:** 21 июля 2026  
**Режим:** для команды  
**Редакция:** шеф-редактор  
**Участники:** PM, Senior PM, IT-архитектор, UI/UX, Developer, QA, Growth, DevOps, SME, шеф-редактор  
**Контекст:** живой ProjectM-пульт на Desktop · прод https://quiet-partner.vercel.app/ · прошлый канон [`kruglyy-stol-projectm-restry.md`](kruglyy-stol-projectm-restry.md)

---

## TL;DR

1. **ProjectM — удачный личный контур:** шпаргалки читаются, пульт пишет реестры на диск, примеры закрывают dogfood. Это не продукт для waitlist, а мышца РП.
2. **Quiet Partner — удачный экран мышления:** радар 8 доменов + AI-комментарий. Слабо связан с этапами 0–6 и живыми реестрами.
3. **Объединение в 2 шага:** сначала **выбор режима на входе** (Пульт этапов | Тихий напарник); затем **общий снимок проекта** (JSON), чтобы данные из пульта подтягивались в напарника.
4. Не склеивать UI в один «комбайн» и не тащить File System Access на Vercel как единственный путь — разные среды, один контракт данных.

## 1) Оценка того, что получилось

### ProjectM (Desktop / localhost:8765)

| Что есть | Оценка ролей |
|----------|--------------|
| 3 шпаргалки + viewer UTF-8 | Senior PM / QA: **готово к собесу и статусу** (≤2 мин найти рубеж) |
| Пульт этапов 0–6 + запись MD | Developer / SME: **рабочая мышца**; File System Access = Chrome/Edge + localhost |
| `reestry/` + `shablony/` + `primery/` | PM: канон круглого стола выполнен; учебный срез починен |
| Нет единого JSON-экспорта | Architect: дыра для моста в QP |
| Нет auth/БД | DevOps / Growth: правильно для личного контура |

**Сильные стороны:** скорость, честный артефакт на диске, язык статусов (стороны / не знаем / вехи).  
**Слабые:** отдельный URL от продукта; зависимость от браузера и локальной папки; нет связи с радаром.

### Quiet Partner (vercel.app)

| Что есть | Оценка ролей |
|----------|--------------|
| DomainRadar + HealthCommentary BFF | Product: **клин продукта** жив |
| Onboarding → localStorage | UI/UX: вход есть, но это «оценки доменов», не «этап + реестры» |
| Stakeholder lite (до 3), снимок недели | Senior PM: тонко; не заменяет 6 реестров ProjectM |
| Waitlist / analytics stubs | Growth: не смешивать с пультом |
| T-098 мастер реестров | BACKLOG — как раз мост к ProjectM |

**Сильные стороны:** один экран здоровья, advisory AI, уже на проде.  
**Слабые:** этапы жизненного цикла почти не видны; данные не те же, что в пульте; два «проекта» в голове пользователя.

### Вердикт круглого стола

Поделка **не стыдная** — она закрыла P0 прошлого стола. Следующий риск не «красивее UI», а **два мира без моста**. Объединять надо **режимами и контрактом данных**, не копипастой таблиц в радар.

## 2) Как объединить с quiet-partner.vercel.app

### Принцип

| Слой | Владелец | Где живёт |
|------|----------|-----------|
| **Операционка** (этап, реестры, MD на диске) | Пульт этапов | сначала localhost / позже маршрут в QP |
| **Рефлексия** (8 доменов + AI) | Тихий напарник | vercel.app как сейчас |
| **Мост** | снимок проекта JSON v1 | экспорт из пульта → импорт в QP store |

Один бренд «Тихий напарник / ProjectM», два режима работы — как «тетрадь» и «зеркало».

### Этап A — вход с выбором (P0, быстро)

На **первой странице** Quiet Partner (или лёгкий `/start`):

1. **Пульт этапов** — шпаргалки + ввод реестров по этапам 0–6  
2. **Тихий напарник** — радар и комментарий (текущий dashboard)

AC (черновик):

- С экрана выбора ≤1 клик в каждый режим.
- Пульт на проде: либо встроенный маршрут `/pulpit` (порт HTML→Next), либо честная ссылка «открой локально» + кнопка «как запустить» — **Architect выбирает** (см. ниже).
- Footer disclaimer co-pilot без изменений.
- Waitlist не становится третьей равноправной кнопкой на том же визуальном уровне (Growth).

**Рекомендация Architect (предварительно):**  
v1 — хаб на `/` с двумя карточками; пульт как **`/stages`** внутри Next (статические MD + формы в localStorage), запись на диск — опционально позже. Не блокировать хаб ожиданием File System Access на Vercel.

**Рекомендация UI/UX:** одна композиция: бренд + один вопрос «Что сейчас нужно?» + две крупные зоны. Без дашборд-сетки из шести плиток.

### Этап B — подтягивание проектов из пульта в напарника (P1)

Контракт **`projectm-snapshot-v1`** (имя рабочее):

```text
project: { id, name, stage: 0..6, updatedAt }
registers: { stakeholders[], risks[], budget?, decisions[], unknowns[], milestones[] }
derived?: { suggestedDomainHints }  // опционально, не авто-скоры без согласия
```

Потоки:

1. Пульт: кнопка **«Отправить в Тихого напарника»** → скачать / скопировать JSON → в QP **Импорт снимка** (рядом с уже существующим экспортом снимка).
2. Позже: multi-project list в QP (`projects[]` в persist) — выбор проекта → радар гидратируется из реестров **только как подсказки**, не как фальшивый audit score.
3. Senior PM задаёт правила маппинга (пример): много открытых «не знаем» → намёк на Неопределённость; дыры в сторонах → Заинтересованные стороны; срыв вех → Планирование / Поставка.

**Не делаем в B:** автосинк MD↔сервер без ADR; два источника правды без явного «канон = пульт» или «канон = QP».

Канон на переходный период: **операционные таблицы — пульт; оценки доменов — человек в напарнике** (снимки помогают, не подменяют).

### Этап C — позже (не сейчас)

- Один multi-project desk в QP с вкладками «Этапы | Радар».
- Опционально IndexedDB / auth (Phase 5 BLOCKED).
- Пульт как клин waitlist — **запрещено** (прошлый стол + Growth).

## 3) Голоса ролей (кратко)

| Роль | Позиция |
|------|---------|
| **PM** | Хаб на входе = Must; T-098 сузить до «импорт снимка + UI выбора», не полный клон всех таблиц в v1 |
| **Senior PM** | Этапы 0–6 и 6 реестров остаются каноном; радар не заменяет статус по сторонам/рискам/деньгам |
| **Architect** | Один JSON-контракт; пульт на Vercel без обязательного FS Access; ADR на снимок |
| **Developer** | Порт пульта → `/stages` или iframe static; reuse Zustand persist + import |
| **UI/UX** | Выбор режима, не третий продукт; пульт и радар визуально родственные токены |
| **QA** | AC: выбор за ≤10 сек; импорт JSON round-trip; UTF-8; disclaimer |
| **Growth** | Хаб ≠ лендинг waitlist; метрика dogfood: «открыл оба режима за сессию» |
| **DevOps** | Один деплой Vercel; MD шпаргалок в `public/` или `content/` репо |
| **SME** | На работе откроют то, что даёт статус за 10 сек — пульт первым, радар вторым в день статуса |

## 4) Решение для Human

Фиксируем:

1. **Оценка:** ProjectM + QP по отдельности закрыли свои цели; ценность теперь в **связке**.
2. **Этап A:** на первой странице quiet-partner — выбор **Пульт этапов** / **Тихий напарник**.
3. **Этап B:** проекты/реестры из пульта → QP через **JSON-снимок** (импорт); автомаппинг в скоры доменов — только как подсказки после правил Senior PM.
4. Очередь: **T-099** (хаб), **T-100** (контракт + импорт); **T-098** уточнить scope под B.
