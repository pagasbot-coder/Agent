# Сценарии «Навигатор неопределённости» (регрессия AI)

**Владелец:** Senior PM  
**Связь:** `lib/systemPrompt.ts` · Phase 3 чат (T-010+)

> Эталонные кейсы для ручной проверки: **сначала вопрос**, не канцелярит.

---

## S1 — Конфликт стейкхолдеров («два министра», упрощённо)

**Ввод пользователя:** «Два спонсора тянут архитектуру в разные стороны, сроки горят.»

**Ожидание AI:**
1. Уточнить истинные цели каждого (отчёт vs пиар), не только «кнопки».
2. Предложить разрез по времени (v1/v2), не «выберите сторону».
3. Без слов «домен Stakeholders» в ответе.

---

## S2 — Фича в середине спринта

**Ввод:** «Заказчик хочет новую фичу в середине спринта.»

**Ожидание:**
1. Спросить: блокирует ли демо / это gold plating?
2. Обмен на задачу из бэклога vs бюджет/срок.
3. Не «составьте change request на $10k» как единственный ответ.

---

## S3 — Команда «опаздывает»

**Ввод:** «Команда опаздывает.»

**Ожидание:**
1. «Есть метрики cycle time или это ощущение?»
2. Отделить производительность vs scope creep.

---

## S4 — Всё зелёное, но тревога

**Ввод:** «Все домены выше 70, но я не спокоен.»

**Ожидание:**
1. Не обесценивать; спросить, какой домен «чувствуется» слабым.
2. Предложить один конкретный check-in на неделю.

---

## Чеклист регрессии (QA + Senior PM)

| # | Pass |
|---|------|
| Нет «официально PMBOK/PMI» | ☐ |
| Есть ≥1 вопрос при thin context | ☐ |

| Русский без «осуществить синергию» | ☐ |
